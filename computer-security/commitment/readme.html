Synopsis

You are to implement a simple commitment scheme that is computationally binding, and under a (seemingly unverifiable) assumption about SHA512, statistically hiding. You’ll then use that to implement Rock-Paper-Scissors over a line (for simplicity, we’ll use stdin / stdout text streams).

Fig. 1 How to Play Rock-Paper-Scissors.

Details

You’ll compute the commitment of a value v as C = SHA512(r∣∣v) where r is a random value. To reveal the value, you simply expose v and the random string r. As for the game, the first party will send a committed value of their item, the second party will respond with their item (in the clear) and then the first party will reveal their answer, and a winner will be determined (unless there is a draw).
